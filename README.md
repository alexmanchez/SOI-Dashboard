# Catena — Crypto Portfolio Exposure Dashboard

LP-style crypto-exposure dashboard for investment advisors managing
discretionary portfolios for family-office / institutional clients.
Gives a day-to-day read on what each client is exposed to across the
managers they've committed to.

Live: https://soi-dashboard.vercel.app

## What this is

An investment advisor (outsourced CIO, allocator, etc.) typically
oversees multiple clients, each of whom has committed capital to
multiple fund managers, each of whom runs one or more funds holding
crypto positions. Catena rolls that up and answers:

- What's this client exposed to right now?
- How concentrated are they in any one token / sector / manager?
- If crypto drops 30% today, how much does each client lose?
- What's the MOIC / TVPI / DPI on each commitment?

Data is client-side only (localStorage). No backend, no accounts. Prices
come direct from CoinGecko; token images come direct from CoinMarketCap.

## Folder layout

- `index.html` — page skeleton Vite injects the React bundle into.
- `package.json` / `package-lock.json` — npm dependencies.
- `vite.config.js` / `postcss.config.js` / `tailwind.config.js` — build
  tool configs, nothing day-to-day.
- `.env.example` — copy to `.env` if you want to override API keys.
- `.gitignore` — files Git skips (`node_modules`, `.env`, build output).
- `src/`
  - `main.jsx` — React entry point.
  - `index.css` — Tailwind base.
  - `SOIDashboard.jsx` — the whole app (see section map below).

`node_modules/` is regenerated by `npm install` and is never committed.
`dist/` is the built output — also never committed; Vercel builds its
own copy on every push.

## Section map of `src/SOIDashboard.jsx`

The file's one big module for now (a future refactor will split it into
folders). Top-to-bottom it's:

1. Imports + color palette constants (dark cypherpunk-institutional).
2. CoinGecko / CryptoRank / CoinMarketCap fetch helpers + embedded keys.
3. Data model helpers — `loadStore`, `saveStore`, `seedStore`,
   `emptyStore`, snapshot + rollup utilities.
4. Shared components — `Panel`, `Pill`, `Tab`, `NavButton`,
   `Breadcrumb`, `EditableText`, `TokenIcon`, `LeftSidebar`.
5. Page components — `OverviewTab` (Dashboard), `ExposuresPage`,
   `FundEconomicsPage`, `ManagersTab`, `SOIDetail`, `PositionsTab`.
6. Settings drawer, Import wizard, Position editor modals.
7. The top-level `SOIDashboard` component that ties everything together.

## Run locally

```bash
npm install
cp .env.example .env      # optional: override keys
npm run dev
```

Open `http://localhost:5173`. Vite hot-reloads on every save.

## Deploying

Pushing to `main` on GitHub triggers a Vercel rebuild at
`https://soi-dashboard.vercel.app`. No other deploy step.

## API keys

All three keys ship embedded in the client bundle:

- **CoinGecko** (`EMBEDDED_CG_API_KEY`) — live + historical prices.
- **CryptoRank** (`EMBEDDED_CR_API_KEY`) — backup source for token images
  and market data.
- **CoinMarketCap** (`EMBEDDED_CMC_API_KEY`) — primary source for token
  logos (including animated GIFs for top tokens).

**Heads up:** anything bundled is visible to anyone who views the
deployed page's source. These are demo-tier keys on an internal tool —
fine for now. For a production public deploy, proxy the calls through a
small backend.

Override any of them by setting the matching `VITE_*` env var in `.env`
or in Vercel's environment variables.

## Features

- **Top-nav** — HOME / PORTFOLIOS (dropdown: clients) / MANAGERS
  (cascading: manager → vintage → FoF underlying) / + CREATE / Search.
- **Left sidebar** — Dashboard / Positions / Exposures / Fund Economics.
- **Dashboard** — performance chart, KPI row, compact sector tilt,
  manager/vintage breakdown, Top 5 Liquid Tokens (animated logos), Top 5
  Movers.
- **Exposures** — full sector tilt, liquid/illiquid donut, manager
  breakdown, Top-25 holdings table.
- **Fund Economics** — pooled Committed / Called / Distributions / DPI /
  TVPI / MOIC, plus per-commitment table.
- **Fund detail** — drill into a single manager+vintage for editable
  positions and economics.
- **As-of date** — global date picker for point-in-time valuation.
- **Live prices toggle** — gate CoinGecko calls so iteration doesn't
  burn demo credits.
- **Inline rename** — click a client portfolio title to rename it.
- **Snapshot-based data model** — every manager's fund has one or more
  dated snapshots; client commitments link a client to a specific
  snapshot with their committed / called / distributions.
- **Fund-of-Funds support** — mark any manager as a FoF, add
  sub-commitments to underlying funds, get one-level-deep look-through
  in client-scoped rollups.
- **Excel export** — Fund Economics workbook with per-commitment rows
  and a dedicated FoF Commitments sheet.
